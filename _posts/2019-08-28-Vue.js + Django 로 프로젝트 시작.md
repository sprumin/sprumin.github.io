### SPA ?

Single Page Application 의 약자인데 작동 원리는 아래와 같다.

처음 호출된 HTML 상에서 페이지 이동 시 html 파일을 변경해주지 않고 필요한 데이터만 호출하여 화면을 새로 구성하기때문에 실제로는 페이지의 이동이 일어나지 않는다! 라고 이해하고있다.

새로운 페이지를 호출하는것이 아니라 내부 구조만 새로 렌더링되는것(?) 이기 때문에 속도가 빠른게 장점이다. 

<br/>

### Vue.js ?

위에서 설명한 SPA 기반의 자바스크립트 프레임워크이다.

React, Angular 라는 선택지도 존재했지만 문서화가 잘 되어있고 낮은 러닝커브를 가진것과 동시에 주변 지인분들의 추천이 있었기에 선택하게되었다.

솔직히 말하면 아직 Vue.js 에 대해서 잘 알지 못한다. 하지만 이번 프로젝트를 통해서 Vue.js 랑 친해졌으면 하는 바램이다. 프론트엔드 프레임워크도 하나는 사용할 줄 알아야하니까...



### RuminScheduler

이전 `새로운 프로젝트를 시작하며` 글에서도 어느정도 기재한 내용이지만 보다 프로젝트에 대해 정확히 작성하기 위해 다시 써본다.

해당 프로젝트는 Vue.js + Django 로 개발하는 일정관리 서비스이다.

로그인, 회원가입 기능을 구현하여 회원별로 일정관리를 가능하게 할것이며, 캘린더 형식으로 월별 일정관리를 하고 해당 날짜 클릭시 일별 issue 관리가 가능하게 ( gitlab issue board 참조 ) 할 것이다.

각 일정의 생성, 수정, 보류, 일정완료 등 status 를 띄워주고 Log 페이지에 일별, 주별, 월별 일정들이 어떻게 진행되었고 status 는 어떠한지 오후 7시에 생성된 Log를 이메일을 통해 전송해줄 예정이다.

가능하면 웹앱까지 생각중이다. 기능적인 부분을 생각하다보니 Log 내용을 이메일이 아닌 모바일 알림을 주고싶어 네이티브앱이나 하이브리드앱으로 개발하면 좋겠지만 이번 프로젝트 목표는 Vue.js + Django 사용이지 어플리케이션 개발이 아니기에 이메일 전송 기능으로 만족해야한다.
