# 새로운 프로젝트 시작

지난 LoveSeulgi 프로젝트가 마무리되고 새로운 프로젝트 아이디어를 생각하던중 회사 선임님이 얘기해주셨던 Vue.js 에 대해 관심이 생겼다. 그렇다고 마냥 단순히 Vue.js 로만 개발하기엔 큰 의미가 없는거같아 Django 를 백엔드로 두고 같이 사용해보기로 했다.

<br/>

### LoveSeulgi 프로젝트에 대해서

처음 기획한대로 결과를 만들어낸 첫 개인프로젝트이기에 뿌듯한감도 있지만 너무나도 많은 부분이 부족하였다.

1. Database 설계
   - 기능별로 app 을 나눠 설계한것까진 잘했다고 생각했다.
   - 그러나 기능을 추가하면서 ForeignKey, ManyToManyField 사용으로 스키마가 꼬여버렸다.
   - 결국 너무 많은 기존기능을 수정해야했고 그에 따라 시간이 낭비되었다.
2. Web Design
   - 항상 느끼는 부분이지만 화면 설계에 대한 감이 부족하다.
   - loveseulgi.kro.kr 을 보면 알겠지만 필요한 기능은 구현이 되었으나 너무 밋밋하다.
3. FrontEnd
   - 처음엔 LoveSeulgi 프로젝트에 Vue.js 를 프론트로 두려고했다.
   - 저떈 Django 기본 이해도도 부족하였고 SPA 의 작동원리도 모르고있었기에 이해도가 부족한 두 프레임워크를 사용하여 진행하게된다면 분명히 도중에 폐기될것 같았다.
   - 그래서 Django 의 jinja2 템플릿만 사용하여 개발하였다.
4. 블로그 포스팅
   - 크롤러에 대한 내용은 크롤러 구현( 초기단계 ) 후 바로 업로드하였지만 그 이외에 것들은 프로젝트 진행이 느려질까봐 프로젝트 종료 후 한번에 포스팅하려고 하였다.
   - 막상 프로젝트가 끝나고 보니 이번 프로젝트에선 부족한점이 너무나도 많았고 굳이 기억해둘만한 이슈가 발생한게 몇 건 없었기 때문에 LoveSeulgi 프로젝트 에 대해 포스팅을 더 이상 안하기로 했다.
   - 만약 LoveSeulgi 프로젝트에 대해 포스팅하게 된다면 버그 수정 중 발생한 이슈일것이다.



### 신규 프로젝트에 대해서

1. 신규 프로젝트?
   - LoveSeulgi 프로젝트를 리뉴얼하는것이 목표이다.
   - 프론트엔드는 Vue.js 로, 백엔드는 DRF 를 이용한 API 서버 형태로 갈아엎을것이다.
   - 회사에 프론트엔드 개발자라는 직군이 따로 없었기에 프론트엔드 개발자와의 협업경험도 없고, Django 를 숙지하는것이 우선이었기에 React/Angular 등의 프론트엔드 프레임워크를 사용해본 경험도 없다.
   - 그렇기에 이번 Vue.js + Django 프로젝트에서는 얻어갈게 많을거같다는 생각이 든다.
2. 프로젝트 목표
   - SPA 에 대한 이해
   - Vue.js 에 대한 이해
   - 프론트/백엔드 각각의 역할 및 개발 방식 이해
   - 주( week ) 단위로 Issue 를 설정해놓고 반드시 개발을 마칠 것 ( 최소 4-5개 )
   - Django Rest Framework 이해
   - Database schema 설계 잘하기
   - 기능별로 나누어 docker-compose 로 배포
3. 블로그 포스팅
   - 이번엔 개발 SPA, Vue.js 이해 과정을 위주로 포스팅할 예정이다.
   - 개발하면서 기억해야하는 부분은 이해한 후 즉각 포스팅하는것이 목표이다.
   - 지난 프로젝트 처럼 닥치는대로 포스팅하는것이 아닌 내가 필요하고 알아둬야할 부분만 포스팅 할것이다.
