<br/>

### 왜 포스팅 안했어요

- docker-compose 관련해서 포스팅하기엔 공부가 더 필요하다고 생각했음요. 해당 지식이 부족하달까
- LoveSeulgi 프로젝트
  - 처음엔 기능 구현하면 그 기능에 대한 포스팅을 바로바로 작성했는데 안그래도 진행속도 느린 프로젝트를 포스팅까지 해가면서 하려니까 이번 생에 못 끝낼거같단 생각이 들었음
  - 프로젝트 끝나고 배포하고나면 포스트 와르르 쏟아질거임



<br/>

### LoveSeulgi

기능은 다 되었는데 웹 디자인하는데 한 세월 걸리는 중.. 항상 말하지만 css 잘하는사람이 세상에서 제일 신기함



#### 진행 상황

- Backend
  - Post, Gallery 삭제 기능 구현해야함 ( 프론트 하다가 안 만들었단걸 깨달음 )
  - Crawler 소스 수정 ( 하단에 기재 )
- Frontend
  - base template & signin template 만 완료되어있음.
  - 솔직히 signin template 도 어떻게 해야 이쁠까 고민중



#### Crawler

수정해야할 부분은 총 두가지이다.

1. Image Size 를 지정하여 크롤링하기
2. Window 에서 image 필터링 후 docker 컨테이너에서 자동 다운로드

각각을 설명하자면 아래와 같다.

<br/>

##### Image Size 지정 크롤링

- 기존엔 이미지의 사이즈는 상관없이 내가 원하는 값이 맞는지만 필터링해서 크롤링했는데 사이트 메인 페이지에서 사용하려고 보니까 각각 사이즈가 전부 달라서  이미지를 보기가 불편하였다.
- 각 페이지에서 사용될 이미지 사이즈에 따라 사용할 수 있도록 검색 조건을 추가해야함



##### Window 에서 image 필터링 후 docker 컨테이너에서 자동 다운로드

- 지금 크롤링하는 방식이 이미지를 전부 긁어온 후 이미지가 정상적인지 일일히 필터링해줘야하는데 cli 기반으로 사용중인 AWS ubuntu 서버에선 이미지가 정상적인지 driver.get 으로 확인이 불가능하다.

- 파일을 그때그때 filezilla 로 올려주기는 불편함이 있기에 redis 를 사용할 생각이다.

- 윈도우에서 기존과 같은 방식으로 크롤링한 후 이미지를 저장하는것이 아닌 AWS ubuntu 서버에서 실행중인 redis 에 아래와 같은 형식으로 캐싱한다.

  ```json
  {
    "크롤러실행날짜/시간": {
        "이미지 SOURCE" : ["이미지", "제목", "출처"],
        "이미지 SOURCE" : ["이미지", "제목", "출처"],
        "이미지 SOURCE" : ["이미지", "제목", "출처"],
                        .
                        .
                        .
        "이미지 SOURCE" : ["이미지", "제목", "출처"],
    }
  }
  ```

- 이후에 AWS ubuntu 서버에서 redis 에 접근한 후 가장 최근에 다운로드한 데이터의 "크롤러실행날짜/시간" 을 일정 시간마다 비교하여 다운로드한다. 이에 관한 크롤러는 추가로 개발할 예정이다.

- 이미지의 유효성마저 자동으로 검사해주면 좋겠지만 거기까진 능력이되지않..
